# 直播创意分析 Skill 使用说明

`livestream-competitor-monitor-skill` 用于把直播间链接、录屏文件、录音文件、转写文本和评论 JSONL 变成结构化的竞品直播分析报告。典型用途是观察抖音直播间的主播话术、直播间贴面、游戏画面、评论互动、福利转化路径，并沉淀可复用的直播创意与话术模板。

## 适用场景

- 输入抖音直播间链接，录制直播画面和系统声音。
- 只录制直播系统声音，用于话术转写和文本分析。
- 对已有录屏或录音文件做分析。
- 同步抓取直播评论区内容，补充评论区互动和观众反馈。
- 生成单个直播间分析报告。
- 对多个直播间内容做对比分析。
- 将分析报告发布为飞书文档。

## 使用前准备

1. 安装 skill 后重启 Codex / Agent 工具，让新 skill 生效。
2. 确认 Windows 上可用 `ffmpeg`。
3. 确认 Python、Playwright Chromium、FunASR 依赖已安装。
4. 录直播系统声音前，需要配置系统声音采集设备，例如 Stereo Mix、virtual-audio-capturer、VB-CABLE、Voicemeeter 或 OBS audio。
5. 如需发布飞书文档，需要安装并登录 `lark-cli`。

注意：当前自动录屏/录音脚本主要面向 Windows。Mac 可以用于已有视频、音频、文本和评论文件分析，但直播间链接自动录屏能力需要另行配置 Mac 录屏和系统声音采集方案。

安装验证里如果看到 `editdistance` 跳过，不代表安装失败。`editdistance` 只用于 WER/CER 文本评测指标，不影响 FunASR 转写、直播分析或报告生成。系统声音采集显示“manual setup needed”也是预期状态，因为它依赖用户本机的音频设备或虚拟声卡配置。

## 快速开始

安装完成后，对 AI 发送直播间链接和模式：

```text
https://live.douyin.com/直播间ID?anchor_id= 录屏
```

只录系统声音时发送：

```text
https://live.douyin.com/直播间ID?anchor_id= 录音
```

停止录制时发送：

```text
停止
```

生成单个直播间分析报告时发送：

```text
生成飞书分析文档
```

如果只想先生成本地 Markdown 报告，可以说：

```text
现在进行单个直播间分析，先不要生成飞书文档
```

## 录屏模式

录屏模式会采集：

- 当前桌面画面
- 系统声音
- 直播间评论 JSONL
- 直播间基础信息
- 自动转写文本

推荐指令：

```text
https://live.douyin.com/直播间ID?anchor_id= 录屏
```

录制开始后，AI 会返回直播间名称、游戏产品名、归档目录、录屏文件路径、评论文件路径和进程状态。

## 录音模式

录音模式只采集系统声音，不录画面。适合重点分析主播话术、节奏、福利引导和评论互动。

推荐指令：

```text
https://live.douyin.com/直播间ID?anchor_id= 录音
```

停止后会生成音频文件、FunASR 转写 Markdown、FunASR JSON，以及评论 JSONL。

## 文件归档规则

默认归档目录：

```text
桌面/直播录文件存档
```

单个直播间会按下面格式建文件夹：

```text
<游戏产品名>-<直播间名称>-<yyyyMMdd>
```

常见文件包括：

- `<直播间名称>-<yyyyMMdd-HHmmss>.mkv`：录屏文件
- `<直播间名称>-<yyyyMMdd-HHmmss>.wav`：音频文件
- `<直播间名称>-<yyyyMMdd-HHmmss>-funasr.md`：转写文本
- `<直播间名称>-<yyyyMMdd-HHmmss>-funasr.json`：转写 JSON
- `<直播间名称>-<yyyyMMdd-HHmmss>-comments.jsonl`：评论 JSONL
- `live-monitor-session.json`：本次录制的 session 信息

## 分析报告内容

单个直播间报告通常包含：

- 直播间基本信息
- 录屏文件、音频文件、转写文本、评论 JSONL 地址
- 一句话结论
- 主播人设与表达方式
- 直播间贴面与 UI 观察
- 核心直播话术
- 游戏画面内容
- 脚本循环机制
- 评论区互动
- 评论区互动洞察
- 本游戏可参考的话术
- 建议测试方向
- 全量主播话术或摘要

UI 关键截图默认不生成。如果需要截图证据，可以单独告诉 AI：

```text
这次报告需要补充 UI 关键截图，我会手动确认截图位置
```

## 多直播间对比

当你已经录制或分析多个直播间后，可以要求对比：

```text
请对最近三个直播间内容做对比分析，输出一份对比报告
```

对比报告会关注：

- 不同直播间的主播人设差异
- 话术循环差异
- 福利和转化入口差异
- 评论区反馈差异
- 更适合本游戏复用的创意策略

## 飞书文档发布

如果已经安装并登录 `lark-cli`，可以让 AI 发布飞书文档：

```text
把这份报告生成飞书文档
```

飞书文档标题默认格式：

```text
<游戏产品名>-<直播间名称>-<日期>
```

发布完成后，AI 会返回：

- 飞书文档链接
- 本地 Markdown 报告路径
- 截图插入是否成功

## 常见问题

### 录不到系统声音

通常是系统声音采集设备没有配置好。请先安装或启用 Stereo Mix、virtual-audio-capturer、VB-CABLE、Voicemeeter 或 OBS audio。不要用麦克风代替系统声音。

### 安装验证里 editdistance 显示跳过

这是可选组件，不影响转写和报告。它只用于 WER/CER 这类文本评测指标。某些较新的 Python 版本没有现成 wheel，会要求 MSVC 编译工具；本 skill 默认不要求用户为这个可选指标安装编译环境。

### 安装验证里系统声音采集显示手动配置

这是预期状态。系统声音采集依赖 Windows 音频设备或虚拟声卡，不能保证在每台电脑上自动安装。只要录屏前配置好 Stereo Mix、VB-CABLE、Voicemeeter、virtual-audio-capturer 或 OBS audio，就可以正常录直播系统声音。

### 直播间名称识别不完整

可以在开始录制时手动指定：

```text
用直播间名称「完整直播间名称」、游戏产品名「游戏名」录屏这个链接：https://live.douyin.com/...
```

### 评论区没有数据

可能原因包括页面未登录、评论区没有新评论、直播页面结构变化、浏览器自动化被拦截。评论抓取失败不会影响录屏，报告会优先使用转写和画面内容。

### 生成飞书文档失败

先确认 `lark-cli` 已安装并登录。如果 PowerShell 执行策略拦截了 `lark-cli.ps1`，AI 会优先使用 `lark-cli.cmd` 或绕过执行策略调用。

### Mac 能不能用

Mac 可以用于已有录屏、音频、文本和评论文件分析。直播间链接自动录屏当前主要面向 Windows，因为系统声音采集和 ffmpeg 设备参数不同。

## 安全与合规

- 只分析公开、已授权或自有直播材料。
- 不要绕过平台访问控制、登录限制、付费墙或反录制措施。
- 不要长期保存观众个人信息；评论内容用于模式总结时应匿名化。
- 不要把原始录屏、评论和转写全文写入 skill memory，只沉淀可复用的非敏感策略。
